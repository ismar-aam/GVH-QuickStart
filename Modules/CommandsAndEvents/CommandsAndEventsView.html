Commands

Commands provide a way to invoke application functionality without knowing or caring about the origin of its implementation.

  • Geocortex Framework Commands:
    • Live in the CommandRegistry of your Application instance,
    • Are referenced by name like so: "app.commandRegistry.command('{COMMANDNAME}')",
    • May have zero or more implementations registered via a "register" method,
    • Are invoked via an "execute" method,
    • Invoke potentially multiple implementations in the order they were registered, and
    • Are created on the fly, that is, referencing a command through the CommandRegistry will always give you an instance.

Events

Events, in a similar manner to commands, provide a way to react to things that happen in an application without being coupled to the source of the event.

Framework events differ from DOM and dojo events in that they are not global. Framework events exist inside a particular Application instance, allowing multiple Framework applications to co-exist on the same page without interfering with each other's events.

  • Geocortex Framework Events:
    • Live in the EventRegistry of your Application instance,
    • Are referenced by name, for example: "app.eventRegistry.event('{EVENTNAME}')",
    • May have zero or more handlers registered via a "subscribe" method,
    • Have a "once" method for registering one-time only handlers,
    • Are raised via a "publish" method,
    • Have handlers that are invoked in the order they were registered, and
    • Are created on the fly, that is, referencing an event through the EventRegistry will always give you an instance.
